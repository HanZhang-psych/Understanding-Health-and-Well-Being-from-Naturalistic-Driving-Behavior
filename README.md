# Sensing Health and Well-Being from Naturalistic Driving Behavior

This repository contains all analysis code for the manuscript. The pipeline implements a prospective exploratory-confirmatory design: driving data from each participant's first year predict health outcomes at the end of that year (Year 1, exploratory), and the same pairing for the second year forms the confirmatory dataset (Year 2). All scripts read from and write to paths relative to the repository root.

> **Data availability.** The raw LongROAD participant-level data are controlled-access and not included in this repository. The preprocessing scripts are shared for transparency. Researchers who obtain the data from AAA can run the full pipeline in the order described below.

## Run order

Steps 1 and 2 have a strict sequential dependency. Steps 3 and 4 depend on the output of Steps 1 and 2. Step 5 depends on Steps 1–4. The supplement and power scripts are independent and can be run at any point after Step 0.

### Step 0 — Build subject-level datasets

```
Rscript analysis/aggregate_data.R
```

Reads the cleaned monthly driving data and per-interval health files, pairs driving from Interval 0 with health at Interval 1 (Year 1) and driving from Interval 1 with health at Interval 2 (Year 2), and writes:

- `data/y1_subject_level.csv` — 2,990 rows, Year 1 (exploratory)
- `data/y2_subject_level.csv` — 2,990 rows, Year 2 (confirmatory)

Missing data are handled at the analysis level; no rows are dropped here.

---

### Step 1 — Bivariate analysis

**1a. Exploratory (Year 1)**

```
Rscript analysis/y1_bivariate_exploratory.R
```

Runs all 288 bivariate regressions (24 driving variables × 12 health outcomes) on Year 1 data, controlling for demographic covariates. Continuous variables are z-scored; categorical variables are dummy-coded. p-values are adjusted using the Benjamini–Yekutieli (FDR-BY) method across all 288 tests. Writes:

- `analysis/output/y1_bivariate_all.csv` — all 288 estimates
- `analysis/output/y1_bivariate_sig.csv` — pairs surviving FDR-BY correction (p.adj < .05); this file is the entry point for Steps 2 and 3

**1b. Confirmatory (Year 2)**

```
Rscript analysis/y2_bivariate_confirmatory.R
```

Reads `output/y1_bivariate_sig.csv`. Tests only the pairs that survived Year 1, one-tailed in the Year 1 direction, with p-values adjusted using the same FDR-BY method over the reduced family. Writes:

- `analysis/output/y2_bivariate_all.csv` — all 288 Year 2 estimates (for reference)
- `analysis/output/y2_bivariate_sig.csv` — Year 1 survivors with Year 2 estimates, one-tailed p-values, and a `replicated` flag

---

### Step 2 — Multivariate path analysis

**2a. Exploratory (Year 1)**

```
Rscript analysis/y1_multivariate_exploratory.R
```

Reads `analysis/output/y1_bivariate_sig.csv`. For each driving variable with at least one significant Year 1 bivariate association with the 11 non-life-satisfaction health outcomes, fits a lavaan path model in which the driving variable simultaneously predicts all 11 outcomes (ML estimator, listwise deletion, fixed.x = TRUE). The candidate relationship is the driving variable's strongest significant Year 1 bivariate association; the model contrasts it against the variable's relationships with each of the remaining 10 outcomes. p-values are FDR-BY corrected across all contrasts. Writes:

- `analysis/output/y1_multivariate_modelspec.csv` — per-IV candidate DV and model specification (frozen for Year 2)
- `analysis/output/y1_multivariate_contrasts_sig.csv` — FDR-BY-significant Year 1 contrasts, carried forward to Year 2

**2b. Confirmatory (Year 2)**

```
Rscript analysis/y2_multivariate_confirmatory.R
```

Reads the frozen Year 1 specification. Refits the same path models on Year 2 data and tests only the contrasts that survived Year 1, one-tailed in the Year 1 direction, with FDR-BY correction. Writes:

- `analysis/output/y2_multivariate_contrasts_sig.csv` — Year 1 contrasts with Year 2 estimates, one-tailed p-values, and a `replicated` flag

---

### Step 3 — Out-of-time prediction (primary: Ridge regression)

Open and run all cells in order:

```
analysis/prediction_ridge.ipynb
```

Trains Ridge regression models on Year 1 data (regularization strength α tuned by inner 10-fold CV on Year 1 only) and evaluates them on Year 2 data. For each of the 12 outcomes, four models are compared on per-subject squared error in Year 2: a constant-mean baseline, a demographics-only model, a driving-only model, and a full model combining both. Missing predictors are imputed using training-set statistics (mean for numeric, mode for categorical); no imputation is applied to outcome variables. Each of the four paired model comparisons is tested with a one-tailed sign-flip permutation test (10,000 permutations); p-values are FDR-BY corrected within each comparison family across the 12 outcomes. Writes:

- `analysis/output/ridge_errors/<outcome>_per_subject.csv` — per-subject squared errors for each of the four models
- `analysis/output/ridge_summary.csv` — MSEs, paired test statistics, and FDR-corrected p-values for all 12 outcomes

### Step 4 — Out-of-time prediction (sensitivity: Elastic Net)

```
analysis/prediction_enet.ipynb
```

Identical to Step 3 except `ElasticNetCV` (L1+L2 penalty, l1_ratio ∈ {0.10, 0.30, 0.50, 0.70, 0.90, 0.95, 0.99}) replaces `RidgeCV`. Serves as the sensitivity analysis reported in the supplement. Writes:

- `analysis/output/enet_errors/<outcome>_per_subject.csv`
- `analysis/output/enet_summary.csv`

## Design summary

| Feature | Year 1 (exploratory) | Year 2 (confirmatory) |
|---|---|---|
| Driving data | Interval 0 | Interval 1 |
| Health outcomes | Interval 1 visit | Interval 2 visit |
| Bivariate tests | All 288; two-tailed; FDR-BY over 288 | Only Y1 survivors; one-tailed in Y1 direction; FDR-BY over reduced family |
| Multivariate contrasts | Candidate identified; contrasts computed; FDR-BY over all contrasts | Only Y1-surviving contrasts; one-tailed; FDR-BY over reduced family |
| Prediction | Ridge/Elastic Net trained here (α tuned by inner 10-fold CV) | Models evaluated here; permutation tests on paired squared-error differences |

## Missing data policy

| Analysis | Handling |
|---|---|
| Bivariate regressions (`lm`) | Row-wise complete case per fitted model; effective N reported with each result |
| Path models (lavaan) | Listwise deletion per model (`missing = "listwise"`, `fixed.x = TRUE`); per-model N reported with each contrast |
| Prediction (sklearn) | `SimpleImputer` fit on Year 1 training data only, applied to Year 2; subjects with missing outcome excluded from that outcome's train/test |
